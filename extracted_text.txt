OptiRelief: Complete Project Description
AI-First Autonomous Coordination Platform for Disaster Response & Community Aid

1. Executive Summary
OptiRelief is not another NGO dashboard. It is a lightweight, AI-agent-based coordination system that transforms fragmented, paper-based field reports into autonomous rescue actions – working offline, on cheap mobile devices, without requiring headquarters approval for routine decisions.
Unlike existing solutions that simply visualize problems, OptiRelief acts on them. When a volunteer submits a messy voice note saying "We're out of water in Azra village, children are sick," the system extracts the need, creates a task, matches it to nearby skilled volunteers, and drafts a restock request – all without human intervention.
Core Innovation: A swarm of specialized AI agents that handle translation, resource prediction, volunteer matching, and task creation autonomously, while remaining under human supervision for high-stakes decisions.

3. Unique Approach: Agent Swarm Architecture
Instead of a monolithic platform, OptiRelief deploys five lightweight AI agents that work independently and coordinate through a shared task board.
Agent 1: The Listener (Voice-to-Task)
Input: Voice memo, handwritten photo, or typed note in any language
Process: On-device transcription → LLM extraction → confidence scoring
Output: Structured task {type: "water", location: "Azra", urgency: 9/10, quantity: "200L"}
Autonomy: Creates task if confidence > 0.85; flags for review if lower
Agent 2: The Dispatcher (Volunteer Matching)
Input: New task + volunteer pool
Process: Cosine similarity matching on (skills + location + availability + past performance)
Output: Ranked list of 3 best-matched volunteers
Autonomy: Sends SMS/WhatsApp offers automatically; waits for acceptance
Agent 3: The Sentinel (Inventory Prediction)
Input: Daily consumption logs from shelters
Process: Linear regression on burn rate + anomaly detection
Output: "Day X will run out of item Y" alerts
Autonomy: Auto-generates restock request 2 days before depletion
Agent 4: The Translator (Field Report Aggregator)
Input: All unstructured field notes from last 24 hours
Process: Dialect normalization → entity extraction → geo-tagging
Output: SQL table of actionable intelligence
Autonomy: Updates central database without human review
Agent 5: The Guardian (Volunteer Retention)
Input: Volunteer activity logs, task feedback, note sentiment
Process: Churn prediction model (random forest, lightweight)
Output: "Volunteer X likely to quit in 3 days"
Autonomy: Sends re-engagement message or reassigns task

9. Data Flow Example (End-to-End)
text
1. Maria (field volunteer) records voice note on WhatsApp:
   "In Azra village, we have no clean water. Two children have diarrhea. 
    There's a doctor named James who just arrived but he has no medicine."

2. Agent 1 (Listener) processes:
   - Transcribes (Whisper tiny on her phone)
   - Extracts: {water: urgent, medical: children, skilled_person: James}
   - Confidence: 0.92 (high)

3. System auto-creates 2 tasks:
   Task A: "Deliver 200L water to Azra" (priority 9)
   Task B: "Contact Dr. James, provide pediatric meds" (priority 10)

4. Agent 2 (Dispatcher) matches:
   Task A → Ahmed (2km away, has truck, available)
   Task B → Dr. Priya (5km away, pediatrician, available)

5. Both receive SMS: "Urgent: Azra village needs you. Reply 1 to accept."

6. Agent 3 (Sentinel) notices water inventory:
   - Current: 500L
   - Burn rate: 250L/day
   - Predicts empty in 2 days
   - Auto-drafts restock request for 1000L

7. Agent 5 (Guardian) monitors:
   - Dr. Priya has accepted 5 urgent tasks in 3 days
   - Sentiment in her notes: "exhausted but helping"
   - Predicts 65% churn risk in 5 days
   - Sends: "Thank you, Dr. Priya. We're routing a volunteer to assist you."

8. Fatima (coordinator) sees dashboard:
   - 2 auto-created tasks (both high confidence, no review needed)
   - 1 inventory alert (2 days remaining)
   - Hope Score: 7.2/10 (good but declining)
   - Approves restock request (1 click)
